Lesson 8｜工具、記憶、workflow 如何串起來？

Step 1｜Agent 的鐵三角：記憶、工具、工作流

一句話版本： 記憶決定了 Agent「知道自己是誰、做到哪裡」，工具決定了它「手能伸多長」，而工作流決定了這一切的「執行順序與標準」。

1. 記憶 (Memory)：從金魚腦到專業幕僚

普通的 LLM 關掉視窗就失憶。但在我們的架構中，記憶被實體化為檔案：

  • 長期記憶 (MEMORY.md)： 記住奇哥的鐵律、課程的終極目標、不可踩的紅線。
  • 短期/課程記憶 (Checklist)： 記住現在做到第幾步、有哪些 Blockers、下一步該做什麼。

2. 工具 (Tools)

語言模型本來只能產出文字，加上 Tools 後就擁有了物理引擎。例如：

  • exec：能在你的電腦裡下指令（建資料夾、跑程式）。
  • scrapling：能繞過防爬蟲機制，抓取真實外部數據（拒絕幻覺）。
  • write：能直接把整理好的報告存到你的桌面。

3. 工作流 (Workflow)

把記憶和工具串起來的繩子。標準的 Agent 工作流是：

  1. 讀取記憶： 看看我是誰、現在進度在哪。
  2. 分析任務： 決定要用什麼工具。
  3. 執行與驗證： 呼叫工具，拿到結果後檢查有沒有失真。
  4. 更新狀態： 把結果寫回實體檔案或 Checklist，完成閉環。

龐德的見解

沒有記憶的 Agent 是瞎子，沒有工具的 Agent 是殘廢，沒有工作流的 Agent 是一盤散沙。

我們在 BeeBond 打造的，是把這三者鎖死在一個標準的工程框架內：用記憶對齊標準，用工具抓取真相，用工作流確保交付。這才是自動化的本質。
