Step 1｜不動筋骨的精準打擊

一句話版本： Model Editing (模型編輯) 像是一把手術刀，專門切除或修改某個錯誤的記憶；Model Merging (模型融合) 像是把中西醫的能力倒進果汁機打成一杯全能果汁，不用花錢重新訓練。

1. 為什麼需要 Model Editing (模型編輯)？

當你發現模型腦子裡有一個非常具體的錯誤（例如：現任總統是誰），如果你把它拿去 Fine-tuning (微調)，可能會引發「災難性遺忘」——它學會了新總統，卻突然忘記了怎麼寫 Python。Model Editing 就是不重訓，直接用數學方法在它的神經網路裡找到那條記憶神經，把它「改掉」。

2. 什麼是 Model Merging (模型融合)？

你有一個超會寫公文的開源模型 A，和一個超會寫 Python 的開源模型 B。你不想花大錢把它們重訓在一起，你可以直接把兩個模型的權重（數字矩陣）相加平均（例如：A × 0.5 + B × 0.5）。神奇的是，出來的新模型 C 往往能同時具備寫公文和寫 Python 的能力。

3. 這跟我們有什麼關係？

開源模型社群（如 HuggingFace）每天都有上千個這樣被「融合」出來的變種模型。身為架構師，當我們需要在本地跑一個垂直領域（如只負責寫 FCPXML）的專用小模型時，懂這些概念，就能在開源市場找到最適合的「縫合怪」來當小弟。
