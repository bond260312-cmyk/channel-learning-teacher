Step 1｜先把概念講懂

一句話先講懂： Context Engineering，就是替 AI 模型安排它每一輪到底該看到哪些資訊，讓它在有限記憶下仍能把事做對。

為什麼這件事重要？

很多人會以為模型夠強、prompt 寫得漂亮就夠了，但當任務開始有工具、有歷史、有多步驟決策時，真正的問題會變成：下一輪到底該讓模型看到什麼？ 如果這件事做不好，再強的模型也會因為上下文混亂而失真。

核心觀念拆解

  • LLM 只看當下收到的輸入，不會自然保留完整歷史。
  • 工具使用一多，上下文會快速膨脹，不能無限塞回模型。
  • Agent 的工作，是篩選、整理、壓縮、排序後，再把最有用的上下文送回模型。

實際流程

  1. 收到任務
  2. 模型做出工具使用決策
  3. 工具回傳結果
  4. Agent 判斷哪些歷史與結果要保留
  5. 重新組成下一輪上下文
  6. 再交給模型做下一步判斷

容易誤解的地方

  • 它不是單純把 prompt 寫漂亮而已。
  • 它不是把所有資料全部塞進去就比較好。
  • 它的關鍵是「選對資訊」，不是「塞更多資訊」。

龐德的見解

這一課真正要抓住的不是名詞，而是一個管理能力：模型不是因為拿到更多資訊就更強，而是因為拿到更對的資訊才更準。

所以 Context Engineering 的核心，不是寫一個漂亮 prompt，而是像幕僚長一樣，知道什麼該保留、什麼該丟、什麼該現在給、什麼該下一輪再給。

最後一句話

Context Engineering 不是把更多資訊塞給模型，而是把對的資訊在對的時機整理給模型。
